## Description

The scheduling plugin is the operational heart of service-based businesses. It provides a full calendar interface (day/week/month views) with drag-and-drop appointment management, resource-based scheduling (by professional and/or location), a waiting list, attendance tracking, and service execution workflows.

**Value beyond CRUD:** This is NOT an entity CRUD plugin. It is a **workflow engine** built around the appointment lifecycle: creation ŌåÆ confirmation ŌåÆ attendance ŌåÆ execution ŌåÆ completion. The calendar UI (react-big-calendar) with drag-and-drop rescheduling, multi-resource views, and real-time updates cannot be generated by `createCrudPage()`.

The plugin also injects tabs into `clients.detail.tabs` (appointment history) and `staff.detail.tabs` (upcoming appointments).

## Custom Pages

### Agenda Page (`/agenda`)

The main calendar view with:
- **Day view** ŌĆö Columns per professional/location, time slots
- **Week view** ŌĆö Week grid with professional/location columns
- **Month view** ŌĆö Monthly overview
- **Agenda view** ŌĆö Flat list of appointments

Features:
- Drag-and-drop to reschedule
- Click to create new appointment
- Color-coded by service or status
- Filter by professional, service, location
- Sidebar with appointment details on selection

### Appointment Confirmation List (`/agenda/confirmations`)

List of appointments pending confirmation with:
- WhatsApp confirmation status
- Bulk confirm/cancel actions
- Filter by date range

### Attendance Checklist

Service execution panel shown when an appointment starts:
- Mark attendance (arrived, no-show)
- Track products used during service
- Record service revisions
- Mark as completed

## Key Workflows

### Appointment Booking
1. Staff selects time slot on calendar (or client books online)
2. Select client (search or create)
3. Select service(s) and professional
4. Appointment created with status `scheduled`
5. Optional: WhatsApp confirmation sent (if whatsapp plugin active)

### Appointment Lifecycle
```
scheduled ŌåÆ confirmed ŌåÆ in_progress ŌåÆ completed
    Ōåō           Ōåō
 cancelled    no_show
```

### Service Execution
1. Client arrives ŌåÆ mark as `in_progress`
2. Attendance checklist opens
3. Record products used (deducts stock if inventory plugin active)
4. Mark services as completed
5. Appointment moves to `completed`
6. Triggers invoice creation (if financial plugin active)

### Waiting List
1. Client wants a service but no availability
2. Add to waiting list with preferences
3. When slot opens, notify staff
4. Staff contacts client and converts to appointment
